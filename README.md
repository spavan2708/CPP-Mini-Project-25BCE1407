# CoopBank — Mini Banking System (DA2 C++ OOP Project)

A console-based cooperative savings account system compiled to **WebAssembly** and hosted via GitHub Pages.

## 📁 Repository Structure

```
/
├── main.cpp        ← Full C++ source (OOP)
├── index.html      ← Custom banking shell UI (replaces Emscripten default)
├── index.js        ← Generated by em++ (DO NOT edit)
├── index.wasm      ← WebAssembly binary (DO NOT edit)
└── README.md
```

## 🚀 How to Build (Locally)

### Prerequisites
- [Emscripten SDK](https://emscripten.org/docs/getting_started/downloads.html)
- Python 3.x
- Git

### Build Steps

```bash
# 1. Activate Emscripten environment (Windows)
emsdk_env.bat

# 2. Compile
em++ main.cpp -o index.js \
  -s EXPORTED_RUNTIME_METHODS='["ccall","cwrap"]' \
  -s FORCE_FILESYSTEM=1 \
  -s MODULARIZE=0 \
  --shell-file index.html

# Or simple build (generates its own index.html — replace with ours):
em++ main.cpp -o index.html
# Then swap the generated index.html with our custom one.
```

> **Tip:** If `em++` is not recognized, run `emsdk_env.bat` first.

## 🌐 Deploying to GitHub Pages

1. Create a **public** repository on GitHub
2. Upload `index.html`, `index.js`, `index.wasm`, `main.cpp`
3. Go to **Settings → Pages → Source: Deploy from branch → main → / (root)**
4. Wait 2–5 minutes → your live link will be:
   `https://<username>.github.io/<repo-name>/`

|---|---------|

| 5 | Transaction history (last N records, with date/time) |

| 8 | Low balance report (threshold-based) |
| 9 | Persistent file storage (accounts.txt + transactions.txt) |
| 10 | Input validation throughout |

## 🏛️ OOP Design

- **`Transaction`** — models one financial event (id, type, amount, balance, datetime)
- **`Account`** — encapsulates private balance; exposes `deposit()` / `withdraw()` only
- **`BankSystem`** — central controller; manages `vector<Account>` and `vector<Transaction>`, file I/O
